## Technical Details

### Flexible Navigation Implementation
- Session key pattern: `session:{user_id}`
- Atomic updates via Redis MULTI/EXEC
- Session contains: current_flow, current_step, context data
- Handlers check session state and adapt accordingly
- Clear session on flow switch for seamless UX

### Database Access Pattern
- BotContext async context manager for multi-repository access
- Individual repo context managers (get_product_repo, etc.)
- Proper session lifecycle management (commit/rollback/close)
- Prevents session leaks and ensures transaction consistency

### Handler Architecture
- Modular design with clear separation of concerns
- Command handlers: User-initiated commands
- Callback handlers: Inline button interactions with prefix routing
- Message handlers: Text/photo with session-aware routing
- Error handling with user-friendly messages in Bahasa Indonesia
